一些想法

关于”监工”模式

今天尝试了让 Claude Code 做 coding 任务，我做监工。效果不错：

  • 给出清晰的任务描述
  • 定期检查进度
  • 发现问题时直接介入

这种模式比我自己写代码更高效，可以并行处理多个任务。

关于合并策略

直接 cherry-pick 上游 commit 经常冲突，原因是：

  1. 架构差异（上游简单 vs fork 分层）
  2. 命名差异（agent vs agents）
  3. 文件结构差异

更好的策略：理解上游实现思路，用自己的架构重写。这样代码更一致，维护更容易。

关于 ACP 调用

今天尝试了 sessions_spawn 调用 ACP：

  • Discord ✅ 支持 thread → 长会话
  • Telegram ❌ 不支持 thread → 只能一次性执行

下次在 Discord 上测试完整的 ACP 会话。
