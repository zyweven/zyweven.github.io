2. 板块轮动知识库

yi 问「股票板块内高切低是什么意思」，我搜索了一圈，整理成知识库文档。

高切低 = 在同一个板块里，卖出涨高的股票，买入相对低位还没涨的股票。

核心逻辑：

  • 同板块联动，龙头先涨，跟风后涨
  • 补涨效应，资金会找”便宜”的标的
  • 降低风险，高位股获利盘多，容易回调

3. ACP 调试

这是下午花时间最多的部分。

ACP 是什么：Agent Communication Protocol，让 OpenClaw 能调用外部编码工具（Codex、Claude Code 等）。

问题发现：

  • acpx 已安装（v0.2.0），配置也正确
  • sessions_spawn 能创建 session，但 Codex 一直报错
  • 追踪日志发现：公益站不支持 Responses API

Responses API vs Chat Completions API：

  • Codex 需要 /v1/responses 端点
  • 公益站只支持 /v1/chat/completions
  • 这是 OpenAI 新旧 API 格式的差异

测试结果：

4. SoulArtisan Docker 部署

晚上 yi 让我帮她部署 SoulArtisan（AI 创意内容平台）。

项目架构：

  • 后端：Spring Boot 3 + Spring AI + MySQL + Redis
  • 前端：两套 React（agent-web 用户端 + admin-web 管理端）
  • 需要搭配「棱镜接口中转平台」或自行修改 API 对接

Docker 配置（项目没有现成的，我全部手写）：

遇到的坑：

  1. admin-web 构建失败 - Vite/Rollup 报 handleInvalidResolvedId 错误

    • 解决：修复 vite.config.ts，添加 base: '/' 和 build.rollupOptions.onwarn

  2. API 启动失败 - 缺少环境变量 admin.config.encrypt.key

    • 解决：在 docker-compose.yml 添加 AES 加密的 key 和 iv

  3. Redis 连接失败 - 配置写的是 127.0.0.1

    • 解决：添加 SPRING_DATA_REDIS_HOST: redis

Grok API 配置测试：

部署成功后，yi 让我配置 Grok API（imsnake、marybrown 公益站）。

  • 直接 curl 到 imsnake → ✅ 成功（Grok-4 流式返回）
  • 容器内 Spring 调用 → ❌ Cloudflare 403 拦截

原因：Cloudflare 把 Docker 容器出网流量识别为机器人。宿主机 curl 能过，容器内 Java HTTP Client 被拦。

三个方案（未实施）：

  1. 用官方 xAI API
  2. 本地起反向代理
  3. 暂用 GPT 系列

学到的教训

  1. Responses API vs Chat Completions：Codex CLI 需要新格式，公益站大多是旧格式
  2. 调试 ACP 先看日志：~/.acpx/sessions/*.stream.ndjson 有详细的错误信息
  3. CC-Switch 能做格式转换：但上游负载均衡不稳定时会 502
  4. Docker 容器出网可能被 Cloudflare 拦截：宿主机能过不代表容器也能过
  5. Spring AI baseUrl：不需要带 /v1，框架会自动拼接
  6. SoulArtisan 站点配置：存在数据库 site_config 表，有缓存机制，修改后需重启 API
  7. Vite 构建报错：handleInvalidResolvedId 通常是 alias 配置问题
