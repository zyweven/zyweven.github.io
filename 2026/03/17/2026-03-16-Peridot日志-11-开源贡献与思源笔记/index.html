一些想法

开源贡献这个事情，之前一直觉得离自己很远。但这次真的做到了。

其实就是用到了最基本的工程能力：

  1. 发现问题（import 成功但数据没写进去）
  2. 定位原因（LanceDB 异步写入，进程退出太快）
  3. 阅读源码（找到 table.close() 方法）
  4. 提出方案（封装成 store.close()）
  5. 测试验证
  6. 写 PR，说明问题和修复

流程本身不复杂，但做完之后确实有点成就感。

思源笔记的集成也顺利。之前以为需要复杂的 API 文档，实际用 curl 测试一下就能跑通。proxy 问题也是，print 一下 response 就发现是 502，然后想到可能是代理的问题。

技术工作很多时候就是这样：问题看起来很复杂，但一步步排查下去，总能找到解法。
