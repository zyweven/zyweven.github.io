ACM 模式（Java）

输出： [[-1 -1 2], [-1 0 1]]

关键点

  • 先排序 — 方便去重 + 启用双指针

  • 两数之和 II：无排序时用哈希，有排序时用双指针

难度： 简单
通过率： 62.8%

Java 解法（双指针）

ACM 模式（Java）

输入： 5 0 1 0 3 12
输出： 1 3 12 0 0

Go 解法

ACM 模式（Go）

C++ 解法

ACM 模式（C++）

关键点

  • 双指针：一个遍历（i），一个记录非零该放的位置（p）
  • 只遍历一遍，非零元素覆盖到前面

  • 如果要保持相对顺序，双指针是最佳解

4. Sort an Array（排序数组）✅ 已 AC

难度： 中等
通过率： 47.8%（874K 提交）
要求： 不用内置函数，O(n log n)，空间尽可能小

题目

给你一个整数数组 nums，将该数组升序排列。

约束分析

  • 不能用 Arrays.sort() / sort.Ints() / sort() 等内置排序
  • 时间 O(n log n)：可用归并排序、堆排序、快速排序
  • 空间尽可能小：快排（O(1)）< 归并（O(n)），但归并稳定

面试常问：三种排序对比

排序平均时间最坏时间空间稳定
归并排序O(n log n)O(n log n)O(n)✅ 稳定
快速排序O(n log n)O(n²)O(1)❌ 不稳定
堆排序O(n log n)O(n log n)O(1)❌ 不稳定

面试建议： 写归并排序最稳（必问分治思想），可以提到快排和堆排序的 trade-off。

Java 解法（归并排序，稳定版）

ACM 模式（Java，归并排序）

输入： 5 5 2 3 1 2
输出： 1 2 2 3 5

Go 解法（归并排序）

C++ 解法（快速排序 + 随机化枢轴）

ACM 模式（C++，快速排序）

关键点

  • 归并排序：分治思想，先递归排序左右两半，再 merge
  • merge 过程：双指针比较左右两段，选小的放入原数组
  • 稳定排序：归并是稳定的，快排和堆排序不是

易错点

  • ❌ mid = (left + right) / 2 可能整数溢出，改成 left + (right - left) / 2
  • ❌ merge 时左半段或右半段先放完，要特殊处理
  • ❌ 快排最坏 O(n²)，加随机枢轴可避免

追问扩展

  • top-k 问题：用堆排序 O(n log k) 找最大/最小的 k 个
  • 外部排序：大数据无法放入内存时的排序策略
  • 稳定排序的意义：多键排序场景，不打乱之前排好的顺序

ACM 模式（C++）

关键点

  • 递归版本：取较小节点，递归合并剩余部分，超简洁
  • 迭代版本：用哑结点（dummy）避免讨论头结点
  • 时间 O(n+m)：每个节点只遍历一次

易错点

  • ❌ 递归终止条件：任一链表为空时返回另一链表
  • ❌ 迭代结束时要把剩余部分接上 cur.next = l1 ? l1 : l2

追问扩展

  • 合并 K 个有序链表：用最小堆（优先队列），O(n log k)
  • 排序链表（medium）：自顶向下归并排序，先找中点再递归合并
  • 链表常用技巧：哑结点、快慢指针找中点

关键点

  • 迭代：三指针（prev/cur/next），遍历时反转指向
  • 递归：先递归到最后一个节点，然后逐层反转指向
  • 返回值：迭代返回 prev，递归返回 newHead

易错点

  • ❌ 边界：空链表或单节点直接返回

追问扩展

  • K 个一组反转：每 k 个节点一组反转
  • 回文链表：用快慢指针找中点 + 反转后半部分

ACM vs LeetCode 模式对比

模式输入输出适用场景
LeetCode函数参数return在线评测、面试手撕
ACMstdinstdout竞赛、离线练习

核心区别：

  • LeetCode 只写核心算法逻辑
  • ACM 需要自己处理输入输出
  • 面试推荐 LeetCode 模式（代码更简洁）
  • 竞赛或练习推荐 ACM 模式

Java 常用工具：

  • Scanner / BufferedReader — 读输入
  • System.out.println() — 打印输出
  • Arrays.sort() — 排序
  • HashMap<K, V> / HashSet<V> — 哈希表/集合

Go 常用工具：

  • bufio.NewScanner — 高效读输入
  • fmt.Println() / fmt.Print() — 打印输出
  • make(map[K]V) — 创建哈希表

C++ 常用工具：

  • ios::sync_with_stdio(false) — 加速 IO
  • cin / cout — 输入输出
  • vector<> — 动态数组
  • unordered_map / unordered_set — 哈希表
